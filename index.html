Taming the Long-Tail: Efficient Reasoning RL Training with Adaptive Drafter

Qinghao Hu*, Shang Yang*, Junxian Guo, Xiaozhe Yao, Yujun Lin, Yuxian Gu, Han Cai, Chuang Gan, Ana Klimovic, Song Han

MIT × NVIDIA × ETH Zurich × UMass Amherst   ·   *Equal contribution

Abstract

Reinforcement learning (RL) has emerged as a key method for training large language models (LLMs) to solve reasoning tasks. However, RL-based training suffers from a critical bottleneck: the long-tail problem in rollout generation. A small number of slow-to-complete requests hold up the entire GPU cluster, wasting compute.

TLT introduces an adaptive speculative decoding strategy that activates a lightweight drafter model only when active requests drop below a configurable threshold. This targets the long-tail phase where GPUs are otherwise idle.

How TLT Works

1
RL Rollout Begins

A batch of prompts is distributed across GPUs. Each generates a response token-by-token using the policy LLM. Most requests finish quickly, but a small fraction produce extremely long outputs.

2
Long-Tail Detected

As requests complete, GPU utilization drops. When active requests fall below the SD threshold, TLT detects the sparse phase and prepares to activate speculative decoding.

3
Adaptive SD Activates

A lightweight drafter generates multiple candidate tokens per step. The main model verifies them in parallel — accepting correct predictions and rejecting wrong ones. This yields 3–4× token throughput on remaining requests.

4
Parallel Drafter Training

While SD accelerates inference, idle GPU capacity is used to continuously train the drafter on freshly generated RL data. This improves the drafter for free — no extra training cost.

5
Lossless Results

Because SD only accepts tokens that match the main model's distribution, output quality is mathematically identical to standard RL training. The drafter is a pure acceleration mechanism with zero accuracy trade-off.
